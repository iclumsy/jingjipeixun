# 报名材料身份证、户口页与毕业证书裁剪优化设计说明

日期：2026-04-02
状态：设计修订完成，待用户复核
范围：`/Users/Ditto/Documents/jingjipeixun/training_system/services/material_service.py`

## 1. 背景与结论

当前报名材料生成会对身份证、户口页和毕业证书执行自动裁边与透视校正，核心逻辑在 `auto_crop_document` 中统一处理。该实现采用单一路径：

1. 灰度化与高斯模糊
2. Canny 边缘检测
3. 闭运算连接边缘
4. 提取外部轮廓
5. 取第一个四边形轮廓并做透视变换

这套方案对边界清晰、背景简单的大幅文档可用，但对身份证、户口页和毕业证书这三类材料的适配性都不够细。根据当前问题反馈，主要失败模式为：

1. 裁偏，把桌面、拍摄背景或阴影带进结果
2. 户口页内部表格线与页面外边界粘连后被误识别为目标轮廓

毕业证书虽然当前没有单独反馈为高频失败项，但它与身份证、户口页共用同一套裁剪入口，后续如果只对前两者做分型处理，证书会继续停留在旧逻辑上，形成新的行为不一致。

本次优化不引入机器学习模型，也不改变报名材料导出接口，而是在现有 OpenCV 方案内升级为“按材料类型定制检测 + 候选框评分选择 + 分级兜底”的传统视觉方案，以提升稳定性和可维护性。

## 2. 目标

1. 明显降低身份证、户口页和毕业证书在复杂背景下的裁偏概率。
2. 降低户口页内部表格线被误识别为外轮廓的概率。
3. 让毕业证书这类大幅页面类材料也具备单独的裁剪规则，而不是继续依赖通用逻辑。
4. 在检测不可靠时优先保守裁剪，避免错误透视导致结果更差。
5. 保持当前依赖栈不变，继续使用 `opencv-python` 完成识别与裁剪。
6. 让后续参数调优具备可观测性，便于基于失败样本迭代。

## 3. 非目标

1. 不引入深度学习检测模型或外部 OCR 服务。
2. 不修改报名材料输出排版逻辑，仅优化输入图片裁剪质量。
3. 不处理拍摄模糊、严重遮挡、证件缺角等超出当前视觉规则能力范围的问题。
4. 不扩展到个人照片、体检表等其他附件的裁剪策略。

## 4. 当前实现的问题分析

当前实现位于 `auto_crop_document`，被身份证、户口页与毕业证书共用。主要问题如下：

1. 候选选择过于简单。代码在找到第一个四边形后直接返回，实际上没有比较多个候选的质量。
2. 对材料类型没有区分。身份证是比例稳定的小卡片，户口页更接近整页文档且内部有大量表格线，毕业证书则属于页面信息密集但内部结构相对自由的证书类材料，三者不适合共用同一套规则。
3. 透视校正门槛过粗。当前只要求面积占比大于 10%，不足以区分“正确页面边界”和“错误背景轮廓”。
4. 缺少保守兜底。一旦误选到背景轮廓，透视变换会把错误进一步放大。
5. 缺少调参依据。当前没有输出候选框面积比、长宽比或评分信息，失败后难以定位是哪一步失效。

## 5. 方案概览

本次改造将统一裁剪函数拆为三个面向材料类型的入口，并引入共享的候选检测与评分流程：

1. `auto_crop_id_card`
2. `auto_crop_hukou_page`
3. `auto_crop_diploma`
4. `detect_document_candidates`
5. `score_candidate`
6. `crop_with_fallback`

高层流程如下：

1. 对原图进行统一预处理，生成适合寻找外边界的边缘图。
2. 提取多个候选四边形或近似矩形，而不是只取第一个。
3. 根据材料类型对候选进行评分并选择最优目标。
4. 当最优候选达到可信度门槛时执行透视校正。
5. 当可信度不足时改为保守外接矩形裁剪，必要时回退原图。

## 6. 分材料类型设计

### 6.1 身份证裁剪

身份证的特点是：

1. 目标比例相对稳定，接近标准卡片比例。
2. 目标面积通常小于整张拍摄图，但不应过小。
3. 内部文字复杂，但外边界通常比内容更规整。

身份证裁剪规则：

1. 预处理仍保留灰度、模糊、边缘检测和形态学处理。
2. 轮廓提取后保留前若干个面积较大的候选矩形。
3. 对每个候选计算面积比、长宽比、矩形度、直角度、边界清晰度和与图像边缘的距离。
4. 评分时优先奖励接近身份证比例的候选。
5. 如果候选过于贴近整张图片边缘，则降权，避免把桌面边缘或拍摄背景当成证件。
6. 仅当评分和面积比同时达到门槛时执行透视校正，否则改用最小外接矩形裁剪并保留适当边距。

### 6.2 户口页裁剪

户口页的特点是：

1. 页面通常较大，接近整页文档比例。
2. 内部横竖表格线密集，容易在闭运算后与外边界连通。
3. 背景误检常来自桌面纹理、垫板边缘或拍摄阴影。

户口页裁剪规则：

1. 在常规预处理前增加光照归一化，减少阴影对边缘检测的干扰。
2. 增加“表格线抑制”步骤，对细密水平/垂直线进行检测并减弱其影响。
3. 候选评分时降低内部线条密度高但外边界不完整的区域得分。
4. 更重视候选轮廓的外边界连续性、矩形度和面积占比。
5. 对户口页透视校正采用更高门槛；分数不足时优先使用保守矩形裁剪，不强行拉正。

### 6.3 毕业证书裁剪

毕业证书的特点是：

1. 通常是较大的页面类材料，面积占比往往高于身份证。
2. 内部既可能有边框、花纹、印章，也可能没有稳定表格结构。
3. 外轮廓通常明确，但拍摄时容易受桌面边缘、背景布纹和阴影干扰。

毕业证书裁剪规则：

1. 使用与户口页相同的页面类基础预处理，但不默认启用强表格线抑制。
2. 候选评分重点关注面积占比、矩形度、外边界连续性和角点质量。
3. 对接近整张拍摄图边缘的大轮廓仍需降权，避免把桌面或背景板整体裁进来。
4. 当候选质量较高时允许执行透视校正，以提升页面摆正效果。
5. 当候选分数中等时优先使用保守矩形裁剪，保留少量留边，避免把证书边框切掉。

## 7. 候选检测与评分设计

### 7.1 候选生成

将候选生成从“找第一个四边形”改为“找多个候选并排序”：

1. 使用 `cv2.findContours(..., RETR_EXTERNAL, ...)` 获取外部轮廓。
2. 对大轮廓进行凸包与多边形逼近，优先保留四边形。
3. 若轮廓不是四边形但矩形度较高，可使用最小外接矩形生成近似候选。
4. 保留前 `N` 个候选进入评分阶段，`N` 初期可取 5 到 10。

### 7.2 评分因子

每个候选至少计算以下指标：

1. `area_ratio`：候选面积与原图面积之比。
2. `aspect_ratio_delta`：候选长宽比与目标材料参考比例的偏差。
3. `rectangularity`：候选面积与最小外接矩形面积之比。
4. `corner_quality`：四角是否接近直角。
5. `border_margin`：候选与原图边缘的最小距离。
6. `edge_density_on_border`：候选边界附近的边缘连续性。
7. `inner_line_density`：候选内部细长水平/垂直线密度，仅对户口页重点使用。

### 7.3 评分策略

评分策略采用加权和，但按材料类型使用不同权重：

1. 身份证重点关注 `aspect_ratio_delta`、`rectangularity`、`corner_quality` 和 `border_margin`。
2. 户口页重点关注 `area_ratio`、`edge_density_on_border`、`rectangularity` 和 `inner_line_density`。
3. 当候选面积异常大且几乎贴近原图边缘时，即使轮廓规则也应明显降权。
4. 当多个候选分数接近时，优先选择边界连续性更好的候选，而不是面积最大的候选。

## 8. 预处理与表格线抑制

### 8.1 统一预处理

三类材料共享以下基础预处理：

1. 按较长边缩放到统一分析尺寸，降低不同分辨率下参数漂移。
2. 转灰度图并做轻度降噪。
3. 必要时做自适应对比度增强，提高页面边界与背景的分离度。

### 8.2 户口页表格线抑制

户口页在候选检测前增加专门步骤：

1. 使用水平核与垂直核提取细长表格线。
2. 对检测到的表格线图层进行减弱或从边缘图中扣除。
3. 保留页面外框等较粗、较连续的外围边缘。

该步骤的目的不是删除所有内部线条，而是避免内部密集网格在闭运算后主导候选轮廓。

## 9. 透视校正与兜底策略

### 9.1 透视校正门槛

仅当候选满足以下条件时执行 `four_point_transform`：

1. 候选评分达到材料类型对应的高可信门槛。
2. 面积比处于合理区间内。
3. 角点质量满足直角约束。
4. 变换后的宽高未出现异常极端值。

### 9.2 保守兜底

当候选不足以支撑透视校正时，不直接失败，而是分层回退：

1. 首选最优候选的最小外接矩形裁剪，并保留适度留边。
2. 如果候选质量仍不足，则返回原图，由后续排版逻辑进行缩放铺放。

这可以避免把“检测不准”的问题放大成“输出图严重变形”。

## 10. 日志与可观测性

为便于后续基于样本调优，需要在裁剪流程中增加调试信息：

1. 材料类型
2. 原图尺寸
3. 候选数量
4. 最终候选的面积比、长宽比偏差和总评分
5. 最终采用的处理方式：透视校正、保守矩形裁剪或原图回退

日志默认以轻量文本方式输出，避免引入新依赖。

## 11. 实现边界

本次实现只修改 `material_service.py` 内部逻辑，不改变以下接口行为：

1. `process_diploma` 的输入输出路径与合成版式不变。
2. `process_id_cards` 的输入输出路径与合成版式不变。
3. `process_hukou` 的输入输出路径与合成版式不变。
4. `generate_student_materials` 的对外行为不变。

为减少改动风险，建议保留原有 `order_points` 与 `four_point_transform`，仅替换裁剪决策逻辑。

## 12. 验证方式

验证重点围绕用户反馈的两类问题展开：

1. 身份证在木纹桌面、浅色桌面、阴影背景上的裁偏情况。
2. 户口页在内部表格清晰、拍摄稍倾斜时的误检情况。
3. 毕业证书在复杂桌面、页面边框较浅或轻微倾斜时的裁剪稳定性。
4. 检测失败时是否优先保守裁剪，而不是出现明显透视拉歪。
5. 新逻辑是否不会影响已有毕业证书、身份证与户口页导出的排版位置。

建议至少准备以下样本集：

1. 正常身份证正反面样本
2. 背景复杂的身份证样本
3. 正常户口页样本
4. 内部表格线明显、边缘不够清晰的户口页样本
5. 正常毕业证书样本
6. 背景复杂或页面边框较浅的毕业证书样本

## 13. 完成标准

满足以下条件即可视为本次优化达成目标：

1. 身份证裁偏问题较当前实现明显减少。
2. 户口页内部表格线误检问题较当前实现明显减少。
3. 毕业证书在页面边缘检测上的稳定性较当前实现明显提升。
4. 在低置信场景下输出结果更保守，不再频繁出现严重拉歪。
5. 实现保持在现有 OpenCV 技术栈内，未引入额外运行依赖。
6. 后续可以根据日志和失败样本继续调权重与阈值，而无需重写整体流程。
